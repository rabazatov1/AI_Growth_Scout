# AI Growth Scout

AI Growth Scout — это агент для студентов, которые хотят развиваться в IT и не знают, где искать хорошие возможности.

Он принимает обычный запрос на русском языке и подбирает курсы, хакатоны, стажировки и образовательные программы. Например, пользователь может написать: «хочу изучить ML с нуля» или «найди хакатоны по AI для студентов», а агент сам решит, какие источники использовать и что показать в ответе.

## Зачем нужен проект

Когда студент хочет войти в IT, ему приходится искать информацию в разных местах. Курсы лежат на образовательных платформах, стажировки — на сайтах компаний, хакатоны — на отдельных сайтах и в календарях. Из-за этого легко пропустить хороший дедлайн или просто не понять, с чего начать.

Я сделал AI Growth Scout как небольшого навигатора по таким возможностям. Идея в том, чтобы пользователь не искал всё вручную, а написал один запрос. Агент собирает подходящие варианты, ранжирует их и объясняет, почему они могут подойти.

Проект сделан в формате Track A + Track C:

- **Track A** — агент использует внешний API Stepik для поиска живых курсов.
- **Track C** — агент использует локальный структурированный JSON-датасет с курсами, стажировками, хакатонами и программами.

## Что умеет агент

- понимать запрос пользователя на естественном языке
- решать, нужно ли обращаться к Stepik API
- загружать кураторский датасет из JSON
- объединять результаты из разных источников
- ранжировать возможности по релевантности
- объяснять, почему конкретные варианты попали в выдачу
- работать даже без LLM, в правиловом режиме

## Инструменты агента

| Инструмент | Что делает |
|---|---|
| `search_stepik_courses()` | Ищет живые курсы через Stepik API |
| `load_curated_opportunities()` | Загружает ручную подборку из JSON |
| `filter_and_rank_opportunities()` | Ранжирует результаты по релевантности |
| `export_results_to_json()` | Сохраняет результаты в `data/cache/` |

## Умная маршрутизация

Агент сам решает, какие инструменты использовать:

- «курсы по python» → Stepik API + кураторский датасет
- «хакатоны для начинающих» → только кураторский датасет
- «стажировки в Яндексе» → только кураторский датасет
- «с чего начать в ML» → Stepik API + кураторский датасет

Это видно в логах: агент печатает каждый вызов инструмента с параметрами.


## Установка

```bash
git clone https://github.com/rabazatov1/AI_Growth_Scout.git
cd AI_Growth_Scout
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Открыть `.env` и вставить ключ от [Groq](https://console.groq.com) в переменную `OPENAI_API_KEY` (проект использует OpenAI-совместимый клиент, поэтому переменная называется именно так):

```
OPENAI_API_KEY=ваш_ключ_groq
OPENAI_BASE_URL=https://api.groq.com/openai/v1
OPENAI_MODEL=llama-3.3-70b-versatile
```

Если ключа нет — проект всё равно работает в правиловом режиме.

## Запуск

```bash
python -m src.main --query "курсы по машинному обучению с нуля"
```

## Примеры запросов

```bash
python -m src.main --query "с чего начать изучение ML"
python -m src.main --query "хакатоны по AI для студентов в мае 2026"
python -m src.main --query "стажировки в Яндексе и Сбере"
python -m src.main --query "бесплатные курсы python для начинающих"
python -m src.main --query "образовательные программы по deep learning"
python -m src.main --query "курсы NLP" --top-k 3 --save-json
```

## Параметры CLI

| Параметр | Описание |
|---|---|
| `--query` | Запрос на естественном языке |
| `--top-k` | Количество результатов (по умолчанию 5, максимум 50) |
| `--offline` | Пропустить Stepik API, использовать только датасет |
| `--save-json` | Сохранить результаты в `data/cache/` |
| `--serve` | Запустить FastAPI сервер |

## FastAPI

```bash
python -m src.main --serve
# Документация: http://localhost:8000/docs
```

Доступные эндпоинты:

```bash
# Статус
curl http://localhost:8000/health

# Запрос агенту
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"query": "курсы по ML для начинающих", "top_k": 3}'

# Список источников
curl http://localhost:8000/sources
```

## Как работает без LLM

Если `OPENAI_API_KEY` не задан, агент переключается в правиловый режим:

- разбор запроса через ключевые слова и регулярные выражения
- ранжирование по тегам, уровню, дедлайну
- шаблонный текст ответа со списком возможностей

Проект полностью работоспособен без LLM — это важно для демонстрации.
